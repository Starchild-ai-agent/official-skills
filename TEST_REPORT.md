### hyperliquid (10 issues)

   - Impact: Small model receives empty/None, cannot diagnose failure
   - Fix: Replace with: raise ToolError("{skill}/{tool}: {specific_reason}")

2. 🟠 **MIXED_RETURN_TYPES** — `client.py:818`
   - Impact: Tool "ensure_dex_abstraction" returns multiple types: {'other', 'dict'}. Small model cannot predict parse strategy.
   - Fix: Standardize: always return dict on success, always raise ToolError on failure
